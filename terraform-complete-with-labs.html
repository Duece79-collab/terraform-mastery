Lab 1: Deploy Your First EC2

Get hands-on with Terraform basics

🎯 Goal: Deploy an EC2 instance using Terraform

What You'll Learn:

  • Create terraform configuration files
  • Initialize a Terraform project
  • Plan and apply infrastructure
  • Manage cloud resources
  • Understand Terraform workflow
1 Create directory and files
mkdir terraform-ec2 cd terraform-ec2 touch main.tf variables.tf outputs.tf
2 Create main.tf with AWS provider and EC2
terraform { required_providers { aws = { source = "hashicorp/aws" version = "~> 5.0" } } } provider "aws" { region = "us-east-1" } resource "aws_instance" "web" { ami = "ami-0c55b159cbfafe1f0" instance_type = "t2.micro" tags = { Name = "my-first-instance" } }
3 Create variables.tf
variable "aws_region" { type = string default = "us-east-1" } variable "instance_type" { type = string default = "t2.micro" }
4 Create outputs.tf
output "instance_id" { value = aws_instance.web.id } output "public_ip" { value = aws_instance.web.public_ip }
5 Run Terraform commands
terraform init # Initialize terraform plan # Preview terraform apply # Deploy
✅ Success! Your EC2 instance is running
✅ Complete Solution

Q: Walk me through deploying an EC2 with Terraform

A: I create main.tf with AWS provider and ec2 resource, variables.tf for inputs, and outputs.tf for important values. Then I run: terraform init (download provider), terraform plan (preview), terraform apply (deploy).

Q: What does terraform apply do?

A: It compares your config to current state, calculates changes needed, and makes API calls to AWS to create/update/destroy resources. After confirmation, state is updated.

Q: Why use outputs?

A: Outputs expose resource attributes (like public IP) so they can be used by other modules or displayed to users. Essential for composition.
